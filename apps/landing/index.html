02 How it works

Drag in tracks. Tell it the vibe. Hit play.

Slow LLM plans the future; a fast deterministic engine owns the clock. A 30-to-60-second lookahead makes live re-prompting glitch-free.

01

Bring your tracks

Drag a folder in. Each file is analyzed locally: BPM, key, energy, beatgrid. Nothing uploads.

02

Describe the set

"40-minute sunset rooftop, build slow, no vocals after the peak." Plain language, not a knob.

03

It conducts

The LLM emits a cue sheet; a deterministic Web Audio engine plays it, beatmatched and harmonic.

04

Steer it live

Tap a quick-prompt or let the mic energy meter read the room. The edit lands at the next phrase boundary, no dropout.

03 The machine voice

A look at the cue sheet.

The conductor doesn't touch a sample. It writes a bar-addressed, versioned JSON cue sheet: the contract between the LLM and the audio engine.

It's typed, small, and hackable. Re-prompt the set and only the future is rewritten; the playing bar is frozen. Read it, fork it, export it.
